title: Définitions
---

Voici quelques définitions pour mieux comprendre le lexique de GRR.

## Glossaire

### Ressource
Une ressource est un élément qui peut être réservé, par exemple :
- une salle ;
- un véhicule ;
- un équipement ;
- une personne.

### Ressource restreinte
Une ressource restreinte est une ressource dont l'accès est limité.  
Par défaut, elle n'est pas accessible à tous les utilisateurs et nécessite des droits ou privilèges spécifiques pour pouvoir être consultée ou réservée.

### Domaine
Un domaine est un ensemble de ressources.  
Il permet de gérer des droits différents et/ou des affichages spécifiques selon les utilisateurs.

### Domaine restreint
Un domaine restreint est un domaine qui, par défaut, n'est pas accessible aux utilisateurs.

### Réservation
Une réservation s'applique à une ou plusieurs ressources sur une période donnée.

### Réservation modérée
Une réservation modérée est une réservation qui nécessite l'approbation d'un gestionnaire avant d'être validée.

### Propriétaire
Le propriétaire d'une réservation est la personne qui crée la réservation.

### Bénéficiaire
Le bénéficiaire est la personne pour qui la réservation est effectuée.  
Il peut être identique ou différent du propriétaire.

### Login / Identifiant
Le login (ou identifiant) est un nom unique par utilisateur permettant :
- la connexion à GRR ;
- la traçabilité des actions.

Il ne peut pas être modifié.

### Statut utilisateur
Le statut utilisateur définit le rôle global de l'utilisateur dans GRR :
- Visiteur
- Usager
- Gestionnaire des utilisateurs
- Administrateur

### Privilèges utilisateur
En complément de son statut, un utilisateur peut posséder des privilèges spécifiques sur certaines ressources, lui accordant des droits supplémentaires.
